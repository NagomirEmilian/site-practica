1. Ce este polimorfismul?

Dacă ești la început cu OOP, sigur ai întâlnit cuvântul polimorfism și poate ți s-a părut complicat. De fapt, ideea e destul de simplă odată ce o înțelegi cu un exemplu concret.

Polimorfism vine din greacă și înseamnă „mai multe forme". În programare, asta înseamnă că un același apel de metodă poate produce comportamente diferite, în funcție de obiectul concret pe care îl ai. Gândește-te la comanda „fa un sunet" — un câine latră, o pisică miaună, o pasăre ciripește. Aceeași comandă, rezultate diferite.

💡 Ideea centrală Polimorfismul îți permite să scrii cod care lucrează cu mai multe tipuri de obiecte fără să știi exact care e tipul fiecăruia. Tu apelezi metoda, iar obiectul știe singur ce să facă.

2. Polimorfism static vs. dinamic

În C# există două forme. E important să le știi pe amândouă.

Polimorfism static — la compilare

Se rezolvă în momentul compilării. Exemplul clasic este supraîncărcarea metodelor (method overloading) — mai multe metode cu același nume dar parametri diferiți, iar compilatorul alege care se apelează.

C# — Static (overloading)
class Calculator
{
    public int    Aduna(int a, int b)       => a + b;
    public double Aduna(double a, double b) => a + b;
    public string Aduna(string a, string b) => a + b;
}

Calculator c = new Calculator();
c.Aduna(2, 3);           // → int: 5
c.Aduna(2.5, 1.1);       // → double: 3.6
c.Aduna("Ion", " Pop");  // → string: "Ion Pop"

Polimorfism dinamic — la rulare

Acesta se rezolvă în momentul rulării programului (runtime). Se bazează pe moștenire și pe cuvintele cheie virtual / override. Compilatorul nu știe dinainte ce metodă va fi chemată — depinde de ce obiect concret există în memorie.

3. Cuvintele cheie virtual și override

Acestea sunt baza polimorfismului dinamic în C#.

  • virtual — în clasa de bază, marchează o metodă ca putând fi rescrisă de clasele derivate.
  • override — în clasa derivată, rescrie comportamentul metodei din bază.
🔑 Regulă importantă Dacă nu pui virtual în clasa de bază, clasele derivate nu pot suprascrie cu adevărat metoda. Pot să o „ascundă" cu new, dar asta e altceva și nu produce polimorfism dinamic.
C# — virtual și override
class Animal
{
    public string Nume;
    public Animal(string nume) { Nume = nume; }

    // virtual = poate fi rescrisă
    public virtual void FaSunet()
        => Console.WriteLine($"{Nume} face un sunet.");
}

class Caine : Animal
{
    public Caine(string n) : base(n) { }

    // override = rescriem comportamentul
    public override void FaSunet()
        => Console.WriteLine($"{Nume}: Ham ham!");
}

class Pisica : Animal
{
    public Pisica(string n) : base(n) { }

    public override void FaSunet()
        => Console.WriteLine($"{Nume}: Miau!");
}

4. Exemplu complet — magia polimorfismului

Acum vine partea interesantă. Putem pune obiecte de tipuri diferite într-o listă de tip bază și apela aceeași metodă pe fiecare — fără niciun if sau switch.

C# — Exemplu complet
class Program
{
    static void Main()
    {
        // Lista de tip Animal, dar obiectele sunt de tipuri diferite
        Animal[] animale = {
            new Caine("Rex"),
            new Pisica("Whiskers"),
            new Caine("Buddy"),
            new Animal("Misterios")
        };

        // Același apel → rezultate diferite. Asta e polimorfismul!
        foreach (Animal a in animale)
            a.FaSunet();
    }
}

/* Output:
   Rex: Ham ham!
   Whiskers: Miau!
   Buddy: Ham ham!
   Misterios face un sunet.
*/

C# verifică la rulare ce tip real are fiecare obiect și apelează metoda potrivită automat — asta se numește dispatch dinamic.

5. Clase abstracte

Uneori nu are sens să poți crea un obiect din clasa de bază. Un „Animal" generic nu există cu adevărat — există câini, pisici, păsări. Aici intră clasele abstract.

O clasă abstractă nu poate fi instanțiată direct. Metodele abstracte nu au corp — clasele derivate sunt obligate să le implementeze.

C# — Clasă abstractă
abstract class Forma
{
    public abstract double Arie();
    public abstract double Perimetru();

    // Metodă normală — moștenită de toate clasele derivate
    public void Descrie()
        => Console.WriteLine($"Arie={Arie():F2}, Perim={Perimetru():F2}");
}

class Cerc : Forma
{
    double _r;
    public Cerc(double r) { _r = r; }
    public override double Arie()      => Math.PI * _r * _r;
    public override double Perimetru() => 2 * Math.PI * _r;
}

class Dreptunghi : Forma
{
    double _l, _h;
    public Dreptunghi(double l, double h) { _l = l; _h = h; }
    public override double Arie()      => _l * _h;
    public override double Perimetru() => 2 * (_l + _h);
}

// Utilizare:
Forma[] forme = { new Cerc(5), new Dreptunghi(3, 4) };
foreach (var f in forme) f.Descrie();
// Arie=78.54, Perim=31.42
// Arie=12.00, Perim=14.00

6. Interfețe

Interfețele sunt similare claselor abstracte, dar mai flexibile. O clasă poate moșteni o singură clasă de bază, dar poate implementa oricâte interfețe.

C# — Interfețe
interface IDesenaabil  { void Deseneaza(); }
interface ISalvabil    { void Salveaza(); }

// O clasă implementează ambele interfețe
class Patrat : IDesenaabil, ISalvabil
{
    double _l;
    public Patrat(double l) { _l = l; }
    public void Deseneaza() => Console.WriteLine($"Pătrat {_l}x{_l}");
    public void Salveaza()  => Console.WriteLine("Salvat în fișier");
}

// Polimorfism prin interfață
IDesenaabil[] obiecte = { new Patrat(4), new Patrat(7) };
foreach (var o in obiecte) o.Deseneaza();

7. Cuvântul cheie base

Când rescrii o metodă, poți apela versiunea din clasa de bază cu base.NumeMetoda() și să adaugi comportament în plus, fără să rescrii totul de la zero.

C# — base
class Vehicul
{
    public virtual void Porneste()
        => Console.WriteLine("Motorul pornește...");
}

class Masina : Vehicul
{
    public override void Porneste()
    {
        base.Porneste(); // apelăm clasa de bază
        Console.WriteLine("Verifică centura de siguranță.");
    }
}

class MasinaElectrica : Masina
{
    public override void Porneste()
    {
        base.Porneste(); // apelăm Masina, care apelează Vehicul
        Console.WriteLine("Activează bateria silențios.");
    }
}
/* Output pentru new MasinaElectrica().Porneste():
   Motorul pornește...
   Verifică centura de siguranță.
   Activează bateria silențios.
*/

8. Greșeli frecvente

Lipsă virtual în clasa de bază

C# — Greșeală vs. Corect
// ❌ GREȘIT — fără virtual, nu e polimorfism dinamic
class Baza    { public void     Metoda() => Console.WriteLine("Baza"); }
class Derivata : Baza { public new void Metoda() => Console.WriteLine("Derivata"); }

Baza obj = new Derivata();
obj.Metoda(); // ← Afișează "Baza"! Nu "Derivata". Surpriză!

// ✅ CORECT — cu virtual și override
class Baza2    { public virtual void   Metoda() => Console.WriteLine("Baza"); }
class Derivata2 : Baza2 { public override void Metoda() => Console.WriteLine("Derivata"); }

Baza2 obj2 = new Derivata2();
obj2.Metoda(); // ← Afișează "Derivata". Corect!

9. Când să îl folosești

Odată ce înțelegi polimorfismul, ai tentația să îl folosești peste tot. Dar există situații unde se potrivește și altele unde complici inutil codul.

Folosește-l când:

  • Ai mai multe tipuri cu comportament comun, dar implementări diferite (animale, forme, vehicule, utilizatori cu roluri).
  • Vrei cod extensibil — adaugi tipuri noi fără să schimbi codul existent.
  • Lucrezi cu colecții eterogene — liste cu obiecte de tipuri diferite.
  • Vrei să elimini lanțuri de if/switch care verifică tipul obiectului.

Evită-l când:

  • Ai un singur tip și știi sigur că nu vei adăuga altele.
  • Ierarhia devine prea adâncă (mai mult de 3–4 niveluri) și greu de urmărit.
  • Comportamentele diferă atât de mult că nu are sens o clasă de bază comună.
📝 Concluzie Polimorfismul dinamic este unul din pilonii OOP. Odată stăpânit, vei scrie cod mult mai elegant și mai ușor de extins. Exersează cu exemple simple — animale, forme, vehicule — și treptat vei vedea cum se aplică în proiecte reale.
